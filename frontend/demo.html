New capability · AI

AI Cost Analyst GPT-4o

CloudLens feeds the full anomaly context — z-score, service-level deltas, resource-group breakdown, trailing 7-day baseline — to GPT-4o and returns a plain-English root-cause explanation with a confidence rating and actionable recommendation. Falls back to deterministic rule-based explanation if no API key is configured.

See live in Insights ↗

✦ AI explanation — 2026-06-08 spend spike

high confidence via gpt-4o · cached: false · 380ms
On 2026-06-08, cloud spend reached €3,420 — 57% above the Holt-Winters seasonal forecast of €2,180 (z-score: 4.2σ). The primary driver was Virtual Machines in subscription prod-westeu, which increased by €890 and accounts for 72% of the spike. A secondary driver is resource group rg-staging, which added €350. The VM increase coincides with a batch job run that was not tagged or scheduled, suggesting unplanned scaling. The staging group increase may reflect a developer left a large dev environment running overnight.
  • Virtual Machines: +€890 (72% of spike) — untagged batch job in prod-westeu
  • rg-staging: +€350 (28% of spike) — likely unscheduled dev environment
→ Check auto-scaling triggers for the batch pipeline in prod-westeu and apply a dev-env shutdown schedule to rg-staging. Estimated monthly saving: €480–€640 if both are addressed.

Autonomous Execution + Terraform Drift NEW

CloudLens can create budgets and alert rules autonomously once a human approves the action. Every resource created autonomously is tagged with CloudLens provenance metadata and a corresponding Terraform HCL snippet is generated — so the infrastructure team can run terraform import to absorb the resource into state and keep the IaC truthful.

🏷 Resource tag convention

"cloudlens:source": "cloudlens-autonomous" "cloudlens:action_id": "act-abc123" "cloudlens:approval_id": "apr-xyz789" "cloudlens:approved_by": "alice@acme.com" "cloudlens:tenant_id": "t-acme" "cloudlens:resource_type": "azure_budget" "cloudlens:created_at": "2026-06-16T09:12:00Z"
