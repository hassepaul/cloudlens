ℹ How it works

CloudLens uses LLM function-calling to map your plain-English question to one of five structured query intents. The LLM only selects and parameterises the query — CloudLens always executes it against Cosmos DB. A second LLM call generates the narrative from the raw results.

top_servicesRanks spend by cloud service
by_cloudCompares across providers
trendDaily spend over a period
comparePeriod-over-period delta
top_resourcesRanks individual resources

Falls back to rule-based intent matching when no OpenAI API key is configured — same API contract, no code changes needed.
