API Reference

Add to .github/workflows/cloudlens-cost-estimate.yml:

name: CloudLens Cost Estimate on: pull_request: paths: ["**.tf", "**.tfvars"] jobs: cost-estimate: runs-on: ubuntu-latest permissions: contents: read pull-requests: write steps: - uses: actions/checkout@v4 - name: Run CloudLens Cost Estimate uses: cloudlens/cost-estimate-action@v1 with: cloudlens_api_url: ${{ secrets.CLOUDLENS_API_URL }} cloudlens_api_key: ${{ secrets.CLOUDLENS_API_KEY }} tenant_id: ${{ secrets.CLOUDLENS_TENANT_ID }} budget_gate_eur: "200" fail_on_gate_breach: "true" working_directory: infra/
Secrets to configure: CLOUDLENS_API_URL · CLOUDLENS_API_KEY · CLOUDLENS_TENANT_ID

In .gitlab-ci.yml:

include: - project: 'your-org/cloudlens' file: 'ci-templates/gitlab-ci.yml' ref: main variables: CLOUDLENS_API_URL: "https://api.cloudlens.io" CLOUDLENS_TENANT_ID: "your-tenant-id" CLOUDLENS_BUDGET_GATE: "200" TF_ROOT: "infra/"
Set CLOUDLENS_API_KEY as a masked CI/CD variable in Settings → CI/CD → Variables.

In azure-pipelines.yml:

steps: - template: ci-templates/azure-devops.yml parameters: cloudlensApiUrl: $(CLOUDLENS_API_URL) cloudlensApiKey: $(CLOUDLENS_API_KEY) tenantId: $(CLOUDLENS_TENANT_ID) budgetGateEur: '200' workingDirectory: 'infra/' failOnGateBreach: true
Add pipeline variables in Pipelines → Library. Mark CLOUDLENS_API_KEY as secret.
POST /api/v1/estimate/terraform/record
Parse plan + persist run. Requires API key.
{ "plan_json": "<terraform show -json output>", "tenant_id": "your-tenant", "label": "PR #42: Add RDS", "ci_system": "github_actions", "repo": "org/repo", "branch": "feature/add-rds", "commit_sha": "abc123", "pr_number": 42, "budget_gate_eur": 200.0 }
GET /api/v1/estimate/runs/{tenant_id}
Pipeline run history. Requires API key. Optional: ?limit=50
POST /api/v1/estimate/gate
Budget gate check. No auth required.
{ "monthly_delta_eur": 150.0, "gate_eur": 200.0, "label": "my-pipeline" }
